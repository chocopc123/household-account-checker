# Kakeibo Matcher: Gemini AIによるマッピング提案機能の実装プラン

家計簿レコードとカード明細のうち、金額の不一致や複数項目の合算により自動マッチングできなかった項目に対し、Gemini AIを用いて推論・提案を行う機能を実装します。バックエンドサーバーは構築せず、フロントエンドアプリから直接APIを呼び出すアーキテクチャに変更します。

## ユーザーレビューが必要な項目

> [!IMPORTANT]
> Gemini APIを使用するために、Google AI Studioなどで発行したAPIキーが必要です。
> ローカル環境での個人利用が前提のため、プロジェクトルートの `.env` ファイルに `VITE_GEMINI_API_KEY=your_key` として設定し、それをアプリから直接参照する方式をとります。

## 提案される変更点

### 1. アーキテクチャ (サーバーサイド排除)

中継用の Node.js(Express) サーバーは構築せず、フロントエンドから直接 Gemini API を呼び出します。

- `@google/generative-ai` SDKを利用するか、直接 `fetch` APIでエンドポイントを叩きます。
- サーバー監視やクロスオリジンの煩雑な設定が不要になり、ローカルアプリとしてよりシンプルに稼働します。

### 2. プロンプト設計

以下のケースに対応できるようなプロンプトを設計し、JSON形式で安定した結果を得られるようにします。

- **店名、内容、日付の推論**: カード明細の「店名」「利用日」と家計簿の「内容」「分類」「日付」から関連性を推測。
- **ポイント支払いの考慮**: 家計簿にはポイント使用前の金額、明細にはポイント差し引き後の金額が記載されているケースの推論。
- **金額の微差**: 税込/税抜の違いや、端数処理による数円の差の許容範囲判定。
- **1:N / M:1 マッチング**: 1つの明細が複数の家計簿レコードに対応する場合などへの対応。

### 3. フロントエンド（レビュー効率化特化 UI）

スライドパネルなどではなく、未マッチ項目の解消（レビューと承認）に特化した、直感的で効率的なUIを導入します。

#### [NEW] [AiReviewDashboard.tsx](file:///c:/Users/choco/develop/src/python_script/householdAccountChecker/src/components/AiSuggestion/AiReviewDashboard.tsx)

- 表の中に埋もれさせず、AIの提案をまとめて処理できる**「一括レビューモード」**画面（または専用ペイン）。
- **カード型レビューUI**: 未マッチ項目に対して、AIが推測した「対象」「理由」「確信度」を1つのカードとして並べて表示。
- **サクサク承認 (Tinder・Inbox風)**: 各カードに「承認(✅)」「却下(❌)」のアクションを配置。**キーボードショートカット (Enter/Esc 等)** に対応させ、大量の提案を次々とさばけるようにテンポの良さを追求します。

#### [MODIFY] [App.tsx](file:///c:/Users/choco/develop/src/python_script/householdAccountChecker/src/App.tsx)

- `@google/generative-ai` を呼び出し推論結果をパースする処理 (カスタムフック `useGeminiAssist` など) を追加。
- 「通常マッチング一覧」とは独立して、「AIによる提案リスト」の状態を管理。
- 提案を承認した場合、即座にメインの照合済みリストへマージさせるロジックの実装。

## 検証プラン

### 自動テスト

- フロントエンドでのAPI呼び出し・レスポンス解析ロジック（`useGeminiAssist`）のユニットテスト。Gemini API のレスポンスをモックし、正常系（妥当なJSON）および異常系（JSONフォーマット崩れ等）のハンドリングを検証します。

### 手動検証

1. `.env` に設定したAPIキーが正しく読み込まれ、AI機能が動作することを確認。
2. 金額がポイント分ずれているダミーデータ等を用意し、AI推論を実行。レスポンスに基づき提案カードが生成されるか確認。
3. カード型レビューUIにおいて、「マッチング理由」がわかりやすく表示されること、およびショートカットキーで高速に承認・却下できることを確認。
4. 最終的に「承認」された項目が正しく集計データやメインテーブルに反映されるかを確認。
